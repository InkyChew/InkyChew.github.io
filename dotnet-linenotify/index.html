.Net 6 - Build Line Notify Web API

Line官方文件中詳細說明需要使用到的API與規格。此篇文章將會透過ASP .NET Web API實作Line Notify，並以ADO .NET Entity Framework的方式存取資料庫。

有關ADO .NET Entity Framework之實做，請參閱Operate database by Entity Framework in .NET。

Implementation

1. Line Notify服務申請

自Line Notify申請服務以取得Client ID、Client Secret，並設定Callback URL。

2. 基本設定

  • 在appsetting中，設定client_id、client_secret、redirect_uri

  • 在controller的建構子中注入依賴的類別

2. 導向Line Notify授權網頁

使用者必須授予服務提供者發送通知的權利，因此第一步便是將使用者導向Line Notify服務授權之頁面。

[HttpGet("{notesId}")]
public IActionResult GetAuthorizeRedirectUrl(string notesId)
{
    var URL = LineAuthorizeAPI;
    URL += "?response_type=code";
    URL += $"&client_id={_lineNotifySetting.Client_ID}";
    URL += $"&redirect_uri={_lineNotifySetting.Redirect_URL}";
    URL += "&scope=notify";
    URL += $"&state={notesId}";
    return Ok(Content(URL));
}

將state設定為使用者的notesId，稍後能透過此notesId將該使用者的AccessToken存放於資料庫。

3. 取得AccessToken

成功授權後，將重新導回服務網頁(redirect_uri)。透過code取得AccessToken，並儲存該使用者的AccessToken至資料庫。

4. 發送訊息

在Header加上Bearer Token發送訊息至Line。
